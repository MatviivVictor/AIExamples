# User Guide

Цей проєкт — консольний AI chat для роботи з кількома провайдерами моделей:
`Ollama`, `OpenAI` і `Gemini`.

## Як користуватися

1. Запустіть застосунок.
2. Виберіть модель зі списку.
3. Введіть текст запиту і натисніть Enter.
4. Дочекайтеся відповіді від моделі.
5. Для виходу введіть `exit`.

Під час звичайного чату програма зберігає історію розмови, тому наступні запити можуть враховувати попередній контекст.

## Які запити можуть бути

Будь-які текстові запити, які ви зазвичай ставите чат-асистенту:

- пояснити поняття або процес;
- допомогти з кодом;
- переписати текст;
- знайти помилку в прикладі;
- запропонувати варіанти рішення;
- коротко підсумувати інформацію;
- відповісти на технічне або загальне питання.

Приклади:

- `Поясни різницю між async і await`
- `Допоможи знайти помилку в цьому фрагменті коду`
- `Перепиши цей текст більш формально`
- `Підкажи, як краще організувати конфігурацію в .NET`

## Що робить `classify`

Команда `classify` запускає окремий режим структурованої класифікації запиту.

Це не звичайна розмова з моделлю. Після введення `classify` програма попросить окремий текст, який треба класифікувати, і поверне машинозчитуваний результат у JSON-подібному форматі.

### Коли використовувати `classify`

Використовуйте `classify`, коли потрібно:

- визначити тип звернення;
- зрозуміти пріоритет;
- оцінити тон повідомлення;
- вирішити, чи треба ескалація до людини;
- підготувати дані для логів, тікетів або маршрутизації.

### Які значення повертаються

Режим класифікації очікує такі поля:

- `category`: `TechnicalSupport`, `Billing`, `Sales`, `GeneralQuestion` або `Unknown`;
- `priority`: `Low`, `Medium`, `High` або `Critical`;
- `sentiment`: `Positive`, `Neutral` або `Negative`;
- `summary`: короткий підсумок одним реченням;
- `shouldEscalate`: `true` або `false`.

### Приклади текстів для `classify`

- `Програма не запускається після оновлення`
- `Не пройшла оплата за підписку`
- `Який тариф підійде для команди з 20 людей?`
- `Поясніть, як налаштувати змінні середовища`

### Коли ставити `shouldEscalate = true`

Система розрахована на ескалацію, якщо:

- критична проблема;
- є ризик втрати даних;
- є проблема безпеки;
- заблокований важливий бізнес-процес;
- запит потребує ручної перевірки.

## Що важливо знати

- `classify` краще вводити окремо, без додаткового тексту.
- Якщо ви введете порожній рядок, програма попросить повторити ввід.
- Якщо введете `exit`, сесія завершиться.
- Відповідь залежить від вибраної моделі та провайдера, але логіка використання однакова.

## Налаштування провайдера

У проєкті вже передбачені три варіанти конфігурації:

- `Ollama` з локальним endpoint;
- `OpenAI` через `OPENAI_API_KEY`;
- `Gemini` через `GEMINI_API_KEY`.

Якщо модель не відповідає або повертає помилку, найчастіше причина в:

- неправильному endpoint;
- відсутньому API key;
- лімітах або квотах провайдера;
- занадто великому запиті;
- тимчасовій недоступності моделі.

## Практичний шаблон використання

Якщо коротко, робочий сценарій такий:

1. Запустили проєкт.
2. Обрали модель.
3. У чаті ввели звичайне питання або `classify`.
4. Отримали відповідь.
5. За потреби повторили запит або завершили сесію через `exit`.
